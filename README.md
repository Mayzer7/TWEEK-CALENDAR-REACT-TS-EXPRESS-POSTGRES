## Функциональность проекта

Минималистичный календарь задач в стиле Tweek: быстрый фокус на дне, неделе и текущих приоритетах без перегруженного интерфейса. Проект сочетает удобное планирование, персональный аккаунт и базовые сценарии ежедневной организации.

- Регистрация и авторизация пользователей с персональным доступом к своим данным
- Управление задачами по дням календаря с быстрым добавлением, редактированием и отметкой выполнения
- Drag-and-drop перенос и переупорядочивание задач внутри дня и между датами
- Поиск задач и быстрый переход к нужной записи или конкретной дате
- Отдельный раздел `Когда-нибудь` для идей и задач без жёсткой привязки ко времени
- Профиль пользователя с возможностью установить аватар
- Восстановление пароля через email для безопасного возврата в аккаунт

## Docker запуск (одной командой)

### Требования
- Docker Desktop (Windows/Mac) или Docker Engine (Linux)

### Запуск
Из корня проекта:

```bash
docker compose up --build --watch
```

После старта:
- **Frontend**: `http://localhost:5173`
- **Backend**: `http://localhost:3001/api/health`
- **PostgreSQL**: `localhost:5432` (user: `postgres`, password: `mayzer`, db: `calendar_db`)

### Что происходит
- `db`: PostgreSQL с уже созданной БД `calendar_db`
- `backend`: Node/Express + автоинициализация таблиц при старте
- `frontend`: Vite dev server

### Остановить

```bash
docker compose down
```

### Сбросить данные БД

```bash
docker compose down -v
```


### Бэкап и восстановление базы данных

Для удобной работы с данными добавлены скрипты автоматизации:

1. **Создать бэкап** (сохранит текущую БД в `backup/calendar_db_backup.sql`):
   ```bash
   npm run backup
   ```

2. **Восстановить базу** (удалит текущую БД и загрузит данные из файла бэкапа):
   ```bash
   npm run restore
   ```

*Примечание: Скрипты должны запускаться из корня проекта при запущенных контейнерах.*

## 🚀 Продакшн (Deployment)

Для запуска проекта в режиме продакшена (максимальная скорость, защита базы, использование Nginx) используйте отдельный конфиг:

### 1. Подготовка
Убедитесь, что в файле `backend/.env` указаны реальные значения для:
- `JWT_SECRET` (сложный ключ)
- `FRONTEND_URL` (ваш домен)
- Данные SMTP для почты

### 2. Запуск
```bash
docker compose -f docker-compose.prod.yml up -d --build
```

### 3. Что изменится в проде:
- **База данных**: Порт 5432 закрыт от внешнего мира (безопасность). Добавлен автоматический перезапуск (`restart: always`).
- **Бэканд**: Код компилируется в JS и работает на чистом Node.js без инструментов разработки.
- **Фронтенд**: Приложение собирается в статику и раздается через высокопроизводительный **Nginx**.
- **Порты**: Фронтенд доступен на стандартном **80 порту**.

### 4. Остановка
```bash
docker compose -f docker-compose.prod.yml down
```
